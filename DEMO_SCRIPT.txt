[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let us look at the architecture of the agent workflow.
First, every query enters at the "security_checkpoint" node, which scans for malicious input and scrubs private details like phone numbers and SSNs.
If the query is safe, the workflow routes it to the "orchestrator" agent.
The orchestrator acts as the lead coordinator, delegating tasks to either the "medication_agent" or the "doctor_agent" based on what the user needs.
These sub-agents use the local MCP Server via stdio to fetch and schedule doctor visits or medication times directly from our database.
Once a schedule is proposed, the workflow halts at the "human_review" node.
Here, a caregiver must manually review and approve the details before they are sent to the patient, ensuring absolute safety.

[3:00 — SECURITY & MCP]
Our security checkpoint runs entirely in Python, using regex for PII scrubbing and structured JSON audit logging to record every safety decision.
Meanwhile, the MCP server hosts four core database tools, keeping data access completely isolated and safe from model hallucinations.
